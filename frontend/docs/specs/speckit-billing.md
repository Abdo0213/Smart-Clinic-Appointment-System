# SpecKit: Billing & Invoices

## Feature Name
Billing — Invoice Management, Payment Status, Invoice Details

## Description
Billing system that manages invoices auto-generated after completed visits. Features include listing invoices, viewing invoice details with line items, marking invoices as paid, and waiving invoices (admin only). Patients can view their own invoices.

## User Stories

- **US-BIL-01**: As a receptionist/admin, I can view all invoices with filters (patient, status).
- **US-BIL-02**: As a receptionist/admin, I can create an invoice manually.
- **US-BIL-03**: As a receptionist/admin, I can mark an invoice as paid.
- **US-BIL-04**: As an admin, I can waive an invoice.
- **US-BIL-05**: As a patient, I can view my invoices.
- **US-BIL-06**: As a receptionist/admin, I can view invoice details with line items.

## UI States

| State | Description |
|---|---|
| **Loading** | Skeleton on invoice list; spinner on payment action |
| **Empty** | "No invoices found"; "No invoices for this patient" |
| **Error** | Toast on payment failure; server error |
| **Success** | Payment confirmed; invoice status updated; waiver applied |

## Data Models

```typescript
type InvoiceStatus = "PENDING" | "PAID" | "WAIVED";

interface LineItem {
  description: string;
  quantity: number;
  unitPrice: number;
  totalPrice: number;
}

interface Invoice {
  id: string;
  visitId: string;
  patientId: string;
  status: InvoiceStatus;
  totalAmount: number;
  lineItems: LineItem[];
  createdAt: string;
  paidAt: string | null;
}

interface CreateInvoiceRequest {
  visitId: string;
  patientId: string;
  appointmentId: string;
  lineItems: Array<{
    description: string;
    quantity: number;
    unitPrice: number;
  }>;
}

interface InvoiceListResponse {
  content: Invoice[];
  totalPages: number;
  totalElements: number;
  size: number;
  number: number;
  first: boolean;
  last: boolean;
  empty: boolean;
}
```

## API Integration

| Action | Method | Endpoint | Auth | Request | Response |
|---|---|---|---|---|---|
| List Invoices | GET | `/billing/invoices` | Admin, Receptionist | `?patientId=&status=&page=&size=` | `InvoiceListResponse` |
| Get Invoice | GET | `/billing/invoices/{id}` | Admin, Receptionist, Patient | — | `Invoice` |
| Create Invoice | POST | `/billing/invoices` | Admin, Receptionist | `CreateInvoiceRequest` | `Invoice` |
| Mark as Paid | PATCH | `/billing/invoices/{id}/pay` | Admin, Receptionist | — | `Invoice` |
| Waive Invoice | PATCH | `/billing/invoices/{id}/waive` | Admin | — | `Invoice` |

## State Management

- **React Query**: `useGetInvoices(filters)`, `useGetInvoice(id)`, `useCreateInvoice()`, `useMarkAsPaid()`, `useWaiveInvoice()`
- **Optimistic updates**: Status changes (paid, waived) can use optimistic UI

## Validation Rules (Zod)

```typescript
const createInvoiceSchema = z.object({
  visitId: z.string().uuid("Invalid visit ID"),
  patientId: z.string().uuid("Invalid patient ID"),
  appointmentId: z.string().uuid("Invalid appointment ID"),
  lineItems: z.array(z.object({
    description: z.string().min(1, "Description required"),
    quantity: z.number().min(1, "At least 1"),
    unitPrice: z.number().min(0, "Price must be non-negative"),
  })).min(1, "At least one line item required"),
});
```

## Edge Cases

1. **Auto-generated invoices**: After a visit is signed, an invoice is auto-generated by the backend. The create form is for manual invoices only.
2. **Patient invoice access**: Patients can only see their own invoices via `GET /billing/invoices/{id}` but there's no `GET /billing/invoices?patientId=X` for patients (only admin/receptionist). Need role-based UI.
3. **Invoice total calculation**: `totalAmount` comes from backend. Client should NOT calculate — but can show running total in create form.
4. **Paid invoice immutability**: Once PAID or WAIVED, invoice should not be editable. Client-side disable actions.
5. **Missing appointmentId in invoice response**: Invoice model doesn't include `appointmentId` in response, but it's required in the create request.
6. **Currency formatting**: Assume clinic currency (EGP based on test data). Format with `Intl.NumberFormat`.
